메모리 주소(Memory Address)

32bit 와 62bit의 차이점
-> 비트의 너비(폭)
-> 직관적으로 데이터 처리 단위

데이터 처리 단위가 많아지면 
CPU처리 고속화

32bit 와 64bit 
-> 메모리와도 연관이 있다.

메모리 한칸은 1byte의 크기를 갖는다
32bit 운영체제는 
-> 4바이트 길이의 주소를 갖는다.

// 집 평수는 1평이고,
이 집을 가리키는 주소는 32자리로
표현된다고 보시면 된다.
(길이와 크기를 혼동하시면 안된다)
-> 길이는 주소
-> 크기는 바이트 수

32bit Architecture
address            low address
0x00000000
0XFFFFFFFF        high address


C언어에서 포인터(pointer)
-> 포인터는 메모리 공간 주소를 
    가리키는 변수
-> 모든 포인터는 
    모두 4byte의 동일한 크기를 갖는다.

포인터는 '주소'를 가리키기 때문에
운영체제가 몇비트이냐에 따라 달라진다

32bit에서의 포인터의 크기는 4byte
64bit에서는 주소의 길이가 8byte
-> 포인터의 크기 도한 8byte일 수밖에 없음
