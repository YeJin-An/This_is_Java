결합도와 응집도

약함 --> 강함
결합도 : 자료, 스탬프, 제어, 외부, 공통, 내용
응집도 : 우연적, 논리적, 시작적, 저라적, 교환적, 순차적, 기능적

=> OPP에서는 응집도는 높히고 결합도는 낮춰야 한다는 말을 많이 하게 된다.
=> 그렇다면 실제 응집도는 무엇이고 결합도는 무엇일까?

응집도란?
-> 하나의 모듈이 하나의 기능을 수행하느 요소들단의 연관성 척도
-> 독립적인 모듈이 되기 위해서는 응집도가 강해야 한다.

= 응집도가 낮은 클래스는?
-> 이해하기 어렵다, 재사용하기어렵다, 유지보수하기 힘들다, 다른 클랫의 변화에 민감하다

= 응집도가 높은 클래스의 특징
-> 단일 책임을 가진 클래스
-> 다른 클래스와 잘 협력하는 클래스

결합도란?
-> 어떤 모듈이 다른 모듈에 의존하는 정보를 측정하는 것이다.
-> 즉, 두 모듈간의 상호 의존하는 정도 or 두 모듈 사이의 연관관계를 이야기 한다.
-> 독립적인 모듈이 되기 위해서는 결합도가 낮아야 한다.

= 결합도가 높은 클래스?
-> 연관된 다른 클래스가 변경되면 더불어 변경해야하고,
-> 수정하려는 클래스를 이해하기 위해 연관된 다른 크래스를 함께 이해해야  하고
-> 재사용하기 힘들다

= 결함도가 낮은 클래스의 특징
-> OCP의 원치을 잘 지킨 클래스 (확장에는 열려있고 변경헤는 닫혀 있는 클래스)
-> 다형성을 잘 지킨 클래스
