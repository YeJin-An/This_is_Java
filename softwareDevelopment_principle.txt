소프트웨어 개발 3원칙

1. DRY(Don't Repeat Yourself)
-> 반복하지 말라
-> 하나하나 수정해야 하므로 버그 발생의 원인이다
-> 소스코드 뿐만 아니라 요구사항, 문자, DB등에 적용될 수 있다.

2. YAGNI(You aren't gonna need it)
-> 필요한 작업만 하자
-> 예측하여 만든 것은 10%정도 밖에 쓰이지 않는다.
-> 필요 이상의 기능은 설계가 복잡하다
-> [ 빨리 구현, 오류 줄이기 ]는 코드를 적게 쓰자.

3. KISS
-> 간단하게 만들자(Keep It Simple Stupid)
-> 짧고 단순하게 만들자(Keep It Short and simple)
-> 작고 단순하게 만들자(Keep It Small and Simple)

// 필요한 코드, 오류줄이기, 단순하고 직관
// 프로젝트를 단순하게 하지 않고, 복잡하게 하면 
   프로젝트를 이해하지 못했다는 증거이다.
